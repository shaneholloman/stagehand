‚Źļ The fix works. Summary of what was needed:

  The solution: CLI click now constructs an Action object and passes it to
  stagehand.act(action):

  const action = {
    selector: resolved,  // xpath
    description: "click element",
    method: "click",
    arguments: [],
  };
  await stagehand.act(action);

  This uses stagehand.act()'s deterministic path (takeDeterministicAction)
  which:
  - Properly resolves the page via awaitActivePage()
  - Handles DOM settling and page state
  - Skips LLM inference (since we already know the xpath and method)

  The key insight: passing an Action object (like observe() returns) to act()
  takes a different, deterministic code path that handles everything correctly.
